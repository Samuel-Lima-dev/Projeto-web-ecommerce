
DOCUMENTAÇÃO DA API - LISTAR ITENS DE UM PEDIDO

Endpoint:
POST /ecommerce/api/index.php?controller=pedido&action=listarItensPedido

Autenticação:
- Tipo: Bearer Token (JWT)
- Cabeçalho obrigatório:
  Authorization: Bearer {seu_token_jwt}

Requisição:
- Método: POST
- Content-Type: application/json
- Body (JSON):
  {
    "pedido_id": 25
  }

Campos esperados:
- pedido_id (integer, obrigatório): ID do pedido que será consultado.

Resposta - Sucesso (200 OK):
{
  "status": "success",
  "itens": [
    {
      "id": 1,
      "pedido_id": 25,
      "produto_id": 7,
      "quantidade": 2,
      "preco_unitario": 49.90,
      "descricao": "Camisa Polo Masculina",
      "total_pedido": 99.80
    },
    ...
  ]
}

Descrição dos campos retornados:
- id: ID do item do pedido
- pedido_id: ID do pedido
- produto_id: ID do produto associado
- quantidade: Quantidade do produto no pedido
- preco_unitario: Preço unitário do produto na época da compra
- descricao: Descrição do produto
- total_pedido: Valor total do pedido

Erros possíveis:

1. 400 Bad Request – Pedido não informado:
{
  "status": "error",
  "message": "ID do pedido não informado."
}

2. 401 Unauthorized – Token ausente ou inválido:
{
  "status": "error",
  "message": "Token JWT inválido ou ausente."
}

3. 500 Internal Server Error – Erro interno:
{
  "status": "error",
  "message": "Erro ao listar itens do pedido: {mensagem do erro}"
}

Observações:
- A requisição exige autenticação via JWT.
- Apenas pedidos do próprio usuário autenticado serão retornados.
